Últimas Ameaças Detectadas

SQL Injection em parâmetro 'id'

Crítica

O parâmetro 'id' na URL é vulnerável a injeção SQL, permitindo acesso não autorizado ao banco de dados.

Recomendação

Implementar prepared statements e validar todas as entradas do usuário. Utilizar ORM para consultas ao banco de dados.

Detalhes Técnicos

Payload testado: 1' OR '1'='1

Endpoint: /products/view?id=1' OR '1'='1

XSS Refletido em campo de busca

Alta

O campo de busca reflete entrada do usuário sem sanitização adequada, permitindo execução de JavaScript malicioso.

Recomendação

Implementar sanitização de HTML no lado do servidor e configurar Content-Security-Policy para mitigar impacto.

Detalhes Técnicos

Payload testado: <script>alert('XSS')</script>

Endpoint: /search?q=<script>alert('XSS')</script>
